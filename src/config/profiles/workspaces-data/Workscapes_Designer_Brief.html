📋 Executive Summary

🔴 The Problem

Workscapes runs expense reports across 4 disconnected systems (Intranet → GlobalSearch → Email → CORE) with no integration. The entire process bottlenecks at AP (Accountant) due to manual GL categorization and manual re-entry into CORE. Managers approve expenses without seeing receipts. Leadership has zero spend visibility.

🟢 The Solution

Replace with a single unified platform: mobile-first submission with AI receipt scanning, inline receipts on approval, GL rules engine for AP, auto-post to CORE via API/RPA, and a company-wide reporting dashboard. Same process — made to actually work.

🎯 Design Mandate

This is not a reinvention — it's the current process simplified. Three moments must feel "magical" in the prototype: (1) snap a receipt and watch fields auto-fill, (2) see receipts inline when approving, (3) see the entire company's spend on one dashboard for the first time.

📱 Mobile Priority

Salespeople, PMs, and field staff need a mobile-first submission flow. Camera capture + AI auto-fill is the flagship UX moment. Desktop is fine for managers (approval), AP (GL review), and executives (dashboard).

📱 Module 1 — Mobile / Web Submission (Employee in field)
📷
Receipt Camera Capture
Take a photo of any receipt. AI extracts merchant, date, amount, and category automatically and pre-populates the form. The "wow" moment of the prototype.
Critical
📎
Multi-File Upload
Support JPG, PNG, PDF. Multiple receipts per single expense. Upload from camera, gallery, or file picker. Eliminates email workaround (PP #9).
Critical
📝
Expense Submission Form
Manager dropdown (always current), expense category selector, description field. AI pre-populates all fields from receipt scan. User can edit before submitting.
Critical
📊
Submission Status Tracker
Employee sees real-time status of their report: Submitted → In Review → Approved / Rejected → Paid. No need to call AP to find out where it stands.
High
🔄
Reject → Resubmit Loop
When a manager rejects with a note, employee receives notification, sees the comment, can correct and resubmit directly. Full audit trail maintained.
High
✅ Module 2 — Manager Approval

Inline Receipt Viewer
Receipts are displayed directly in the approval card — no separate tab, no login to another system. Solves PP #1 and the broken GlobalSearch view (PP #5).
Critical
⚡

Approve in one click. Reject with an optional note (required for reject). Rejected reports trigger resubmit loop to employee immediately.
Critical

🤖
GL Pre-fill (Rules Engine)
System automatically suggests the correct GL code for each expense line based on category rules. Accountant reviews, confirms, or overrides. She stays in control — AI does the heavy lifting.
Critical
🔗
Auto-Post to CORE
Once Accountant confirms the GL breakdown, data auto-posts to CORE via API (Option A) or RPA (Option B). Manual re-entry is completely eliminated — the biggest bottleneck resolved.

⚠️ Open Questions — Resolve Before Customer Review
Gap 1 — Mobile Commitment
Native app (iOS/Android) vs PWA/Responsive Web?
The tech architecture shows "Mobile App (iOS · Android)" but the recommendation is "responsive web as baseline." AP Coordinator explicitly wants a phone-camera flow for field staff. This decision affects development timeline and must be locked before prototype handoff.
Gap 2 — Multiple Receipts Per Line (PP #9)
Is "multiple receipts per expense line" explicitly supported?
Business case requires this. Design shows "+ Add another receipt" but it needs to be explicitly spec'd as multiple files per single expense line, not just one receipt total per report. Affects the upload component design.
Gap 3 — Reject → Resubmit Loop
Full flow for rejection with notes and resubmission not yet documented.
What states does the report enter? Does the manager re-review only the corrected line or the full report? Is there a resubmission limit? This loop needs a screen in the prototype — designers need clarity before building.
Gap 4 — AP Coordinator's Division Rollup Logic
How is the Ops & Procurement rollup hierarchy defined?
AP Coordinator wants to see her managers' direct reports' expenses (full division). The platform says "AP Coordinator sees the rollup" but doesn't specify the role hierarchy model. Without this, the dashboard cannot satisfy PP #2. The Admin module must support configuring this.
Gap 5 — Approval Aging / SLA Definition
What are the exact SLA thresholds and who gets notified?
AP Coordinator mentioned a 3-day approval expectation and a previously-removed 2-week handbook cycle. The dashboard should show aging reports, but we need: (a) what threshold triggers a flag, (b) does the manager get a reminder push/email, (c) does AP Coordinator see a summary of breaches?
Gap 6 — Monthly Category Comparison View
CFO's specific ask: fuel in June vs. dining in March.
The dashboard description says "by category, time period" but doesn't show a monthly comparison chart. Adding a concrete example (side-by-side bar chart, two periods) to the prototype would significantly strengthen CFO's confidence in the solution during the demo.
Gap 7 — CORE Integration Option (A vs B)
API integration (Option A) or RPA (Option B)?
Both options are listed as viable. This doesn't affect the UX prototype directly, but the designer should know whether to show an "integration settings" screen in the Admin panel or not. Recommend noting "auto-post to CORE — integration method TBD" in the prototype.
Gap 8 — Pain Point #9 Explicit Messaging
Receipt upload reliability needs explicit callout in the submission screen.
The fix for email workarounds (PP #9) is only implicitly addressed. The submission screen should prominently display: "Upload from camera, gallery, or file — JPG, PNG, PDF supported — multiple receipts per expense." This is the message that closes this pain point in the demo.
